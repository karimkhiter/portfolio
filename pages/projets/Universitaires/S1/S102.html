Contexte et Mise en Situation

Le Scénario

En tant que consultant intégration de données, la mission est de concevoir un programme complet permettant de lire, transformer et écrire des données de résultats de matchs de football, dans le respect de spécifications strictes garantissant l'interopérabilité avec un système cible. Le programme doit ingérer 14 années d'historique de matchs de Ligue 1 (saisons 2010/11 à 2023/24) et en calculer automatiquement le classement ELO de chaque équipe, saison par saison.

C'est quoi le score ELO ?

Le système ELO est un algorithme de classement inventé par le mathématicien Arpad Elo pour les échecs, aujourd'hui adapté à de nombreux sports. Le principe : chaque équipe possède un score ELO, et chaque match le fait évoluer selon deux facteurs — la probabilité de victoire attendue et le résultat réel. Battre une équipe forte rapporte plus que battre une équipe faible.

📐 Formule ELO utilisée

' Probabilité de gagner pour l'équipe A
P(A) = 1 / (1 + 10 ^ (différenceELO / 400))
' Incrément ELO après le match
ΔElo = K × (résultat_réel − P(A))
' Où K est le coefficient de développement
Const COEF_DEVELOPPEMENT_DEFAUT = 32

Extrait de Code — Algorithme ELO

La fonction centrale du projet : calcul de l'incrément ELO après chaque match.

' Calcule la probabilité qu'une équipe gagne selon la différence ELO
Function CalculerProbabiliteGagner(ByVal differenceElo As Long) As Double
    CalculerProbabiliteGagner = 1 / (1 + 10 ^ (-differenceElo / 400))
End Function

' Calcule l'incrément ELO à appliquer après un match
Function CalculerIncrementElo(ByVal score As Double, _
                            ByVal probabiliteGagner As Double, _
                            ByVal coefDeveloppement As Long) As Long
    ' score = 1 (victoire), 0.5 (nul), 0 (défaite)
    CalculerIncrementElo = coefDeveloppement * (score - probabiliteGagner)
End Function
' Calcule l'année de championnat d'un match selon la date
' (la Ligue 1 se termine en mai → l'année de saison dépend du mois)
Function CalculerAnneeChampionnat(ByVal dateMatch As Long) As Integer
    If Month(dateMatch) < MOIS_FIN_CHAMPIONNAT Then
        CalculerAnneeChampionnat = Year(dateMatch)
    Else
        CalculerAnneeChampionnat = Year(dateMatch) + 1
    End If
End Function

Analyse Réflexive

AC11.03 & AC11.05 | VBA et algorithmique

C'est mon premier projet de programmation structurée. Devoir séparer le code en 6 modules distincts, chacun avec une responsabilité claire, m'a appris ce qu'est la modularité. L'implémentation de l'algorithme ELO a nécessité de traduire une formule mathématique en logique algorithmique : gérer les cas nul/victoire/défaite, les boucles sur les matchs, les mises à jour simultanées des deux équipes à chaque rencontre.

AC11.04 | La problématique du tiret

Un problème concret et représentatif : le tiret dans le score "2-1" dans les fichiers sources n'était pas le tiret du clavier standard mais un caractère Unicode différent. Sans traitement spécifique dans CalculerScoreNormalisé(), toutes les tentatives de parsing échouaient silencieusement. Ce bug m'a fait réaliser que la préparation et la validation des données en entrée sont aussi importantes que l'algorithme lui-même.

👥 Travail en équipe

Ce projet à 4 m'a confronté aux enjeux du développement collaboratif : qui fait quoi, comment partager le code, comment éviter les conflits entre modules. La séparation en modules a facilité la répartition du travail — chacun pouvait développer son module de façon relativement indépendante, à condition de respecter les interfaces définies.

💡 Ce que j'ai retenu

Ce projet m'a démontré concrètement la puissance de l'automatisation : calculer le classement ELO de toutes les équipes de Ligue 1 sur 14 saisons — des milliers de matchs — en quelques secondes là où ça prendrait des semaines manuellement. C'est ici que j'ai compris pourquoi on programme.
