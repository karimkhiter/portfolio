Création d'un Quiz Interactif avec Évaluation par IA (LLM)

🔑 Objectifs Clés

  • Concevoir une architecture scalable séparant la logique métier du contenu brut.
  • Gérer les états d'une application web interactive (Single Page App).
  • Intégrer et brider une IA générative pour la correction automatisée via Prompt Engineering.

Contexte et Mise en Situation

Le Scénario

L'évaluation des connaissances est souvent cantonnée à des QCM binaires car la correction des questions ouvertes demande un temps humain trop important. L'objectif de ce projet suivi était de concevoir de A à Z une application Web d'évaluation interactive. L'innovation majeure : l'intégration d'un Modèle de Langage (LLM) capable de lire la réponse textuelle de l'étudiant, d'en évaluer la justesse technique, et de lui attribuer une note justifiée en temps réel.

Problématique

Comment développer une application d'évaluation qui soit à la fois "Scalable" (personnalisable par un non-développeur), dynamique (interface web), et algorithmiquement intelligente (intégration d'IA pour la correction) ?

Plongée dans le Code : Démarche Technique

Pour répondre à cette problématique complexe, nous avons architecturé l'application en séparant strictement la donnée, l'interface et l'intelligence artificielle.

1. Scalabilité des Données (Le Parseur YAML)

Pour rendre l'application utilisable par n'importe quel professeur, il était hors de question de coder les questions en "dur" dans le fichier Python. Nous avons opté pour une configuration via un fichier .yaml. Le script Python charge dynamiquement ce fichier au démarrage. Le professeur peut y définir le "Prompt Global" (le comportement de l'IA) et les questions ouvertes ou fermées.

Extrait de exemple_quiz.yaml — Le Prompting de l'IA :

quiz:
  titre: "Quiz BUT Science des Données 1A"
  prompt_global: |
    Tu es un professeur expert en BUT Science des Données.
    Tu dois évaluer la réponse d'un étudiant à une question donnée.
    
    Format de réponse OBLIGATOIRE :
    Rédige d'abord un commentaire constructif.
    Sur la toute dernière ligne, indique la note finale ainsi :
    score = X
  questions:
    - type: "question_ouverte"
      question: "Expliquez la différence entre une liste et un dictionnaire en Python."

2. Intégration du LLM (Le Hack OpenAI / HuggingFace)

Pour évaluer la question_ouverte, nous avons connecté notre application à un modèle hébergé sur HuggingFace. L'astuce d'ingénierie a consisté à utiliser la bibliothèque standard openai de Python, mais en redirigeant sa base_url vers le routeur HuggingFace. Cela permet d'utiliser n'importe quel modèle open-source robuste tout en gardant une syntaxe d'API standard et très documentée.

Extrait de app.py — Redirection d'API :

import openai from OpenAI
import re # Expressions régulières pour extraire la note

# --- CONFIGURATION API HUGGING FACE ---
API_KEY = "XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"

client = OpenAI(
    # Redirection de la requête de base vers les serveurs HF
    base_url="https://router.huggingface.co/v1",
    api_key=API_KEY
)

3. Moteur Interactif et State Management (Dash)

L'interface utilisateur (Frontend) et le Backend ont été développés en pur Python grâce au framework Dash et Dash Bootstrap Components. Le véritable défi technique ici était le "State Management" : suivre l'avancée de l'étudiant, sauvegarder ses réponses temporaires, et mettre à jour le DOM sans jamais recharger la page. Nous avons utilisé les @app.callback avec du Pattern Matching pour gérer dynamiquement des boutons générés à la volée.

Extrait de app.py — Le Callback de Navigation :

@app.callback(
    Output("quiz-content", "children"),
    Input({"type": "nav-btn", "action": ALL}, "n_clicks"), # Pattern Matching
    State("store-app-state", "data"),
    State({"type": "answer-input", "index": ALL}, "value"),
    prevent_initial_call=True
)
def navigate_quiz(n_clicks_list, state, current_answer_val):
    # Sauvegarde temporaire de la réponse avant changement de page
    if current_answer_val:
        state["answers"][str(current_q_idx)] = current_answer_val[0]

4. Déploiement "Zéro Configuration"

Un bon code ne sert à rien s'il est impossible à lancer par l'utilisateur final. Pour rendre notre projet totalement "Plug & Play", nous avons rédigé le fichier requirements.txt contenant les librairies (dash, pyyaml, openai, pandas, plotly), couplé à un script batch (.bat) Windows. L'utilisateur n'a qu'à double-cliquer pour que l'environnement s'installe tout seul.

@echo off
echo Installation des dependances Python
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
echo ✅ Installation terminee
pause

Bilan et Analyse Réflexive

Dompter une IA générative

La véritable difficulté de ce projet ne fut pas le Python, mais le "Prompt Engineering". Une IA générative par nature a tendance à "bavarder". Il fallait la forcer à toujours répondre avec un format syntaxique très strict pour que l'expression régulière (Regex) de mon script Python puisse extraire le "score = X" à la fin de sa réponse. J'ai appris qu'une IA intégrée dans un pipeline logiciel doit être fortement bridée.
