Contexte et Mise en Situation

Le Scénario

Le web est une source intarissable de données. En tant que consultant décisionnel, je suis missionné pour concevoir une solution capable de collecter, structurer et qualifier des données produits externes pour alimenter un entrepôt de données.

Objectifs et Problématiques

L'enjeu est double : technique (récupérer la donnée via des API ou du scraping) et structurel (organiser cette donnée pour qu'elle soit exploitable). Il faut garantir l'intégrité des données, respecter l'éthique du web (robots.txt) et automatiser le processus.

Mon Projet : Scraping Blue Buffalo

Dans ce projet de groupe, nous étions responsables de la marque Blue Buffalo. L'objectif était de créer un scraper robuste sur Google Colab utilisant la bibliothèque Playwright pour simuler un navigateur et extraire les fiches produits.

Le Défi Technique

Le site Blue Buffalo présente des défis spécifiques (contenu dynamique). Notre script réalise les étapes suivantes :

  1. Connexion : Navigation vers les pages Chiens et Chats via Playwright.
  2. Extraction : Récupération des balises HTML (Nom, Prix, Ingrédients, Poids, Image).
  3. Nettoyage : Utilisation de strip() et replace() pour nettoyer les textes (ex: retirer les devises, normaliser les poids).
  4. Stockage : Création d'un DataFrame Pandas et export en CSV respectant le schéma imposé (colonnes : brand, species, sku, rating...).

Analyse réflexive par Apprentissage Critique

AC21.01 | Organisation des données

Pour répondre à cet AC, j'ai dû concevoir la structure du fichier de sortie avant même de coder. J'ai analysé les besoins (quelles colonnes pour quelle analyse future ?) et mappé les données brutes du site (ex: "20 lbs") vers des données structurées (valeur: 20, unité: lbs) exploitables en SQL ou Python.

AC21.03 | Intégration hétérogène

Le défi était l'hétérogénéité des pages web. Certaines fiches produits Blue Buffalo avaient un type de class, d'autres un autre; certaines avaient une informtion, d'autres non... J'ai développé des conditions logiques pour gérer ces cas (valeurs NULL) sans faire planter le script d'intégration.

AC21.05 | Briques logicielles

Plutôt que de tout recoder, j'ai choisi d'utiliser Playwright au lieu de Requests car le site utilisait beaucoup de JavaScript (chargement dynamique). J'ai appris à lire la documentation de cette bibliothèque pour gérer les attentes implicites (wait_for_selector).
